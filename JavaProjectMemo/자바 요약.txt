배열 => 고정된 값, int i2 [] = {...};, int i3 [] = new int [1,2,3];
Arrays => 배열 조작 클래스 Arrays.asList(배열) = 리스트로 변환
고정된 값이 싫어서 ArrayList 등장 => 가변적인 배열 길이 가능

상속
구현 하지 않아도 부모의 클래스의 요소를 자식이 가지고 있다고 생각하면 된다.
객체 생성해서 가지고 오는 것이랑 비슷하다.
그러나 메인 메서드에서 자식 클래스만 구현하였을 때 생성자를 통해 값을 넣어주면 자동으로 부모 클래스의 생성자를 
가지고 오는 것은 아니다. super(name) 같은 형식으로 자식에서 구현해줘야 한다.

인터페이스
1. 구현체 = new 구현체
2. 인터페이스 = new 구현체 => 다형성 성질 획득
인터페이스가 인터페이스 상속가능, 구현체에서 2개다 구현해야 함.
구현체가 여러 인터페이스 imple 가능 => 다 구현해야 함.
기본적으로는 비어있는 메서드만 있어야 하지만 자바 버전 업으로 default 메서드를 이용하여 인터페이스에서 일반적인 메서드를 구현할 수 있다. 

추상 클래스
구현체는 extends를 사용하여 구현한다.
추상 클래스의 메서드는 구현체에서 다 구현해야 한다.
추상 클래스에서 비어있는 메서드도 있고 일반적인 메서드도 있다.
