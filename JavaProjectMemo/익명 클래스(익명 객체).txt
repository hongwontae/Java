익명 클래스(익명 객체)

프로그램에서 일시적으로 한번만 사용되고 버려지는 객체, 재사용 x
=> 자바는 확장성을 염두해두고 로직을 생성하는데 단지 한번의 사용만 할 객체를 클래스로 만들어버리면 오히려 확장성이 효율성을 해치는 행위가 된다.

익명 클래스를 구현하는 방법
1. 익명 자식객체 (부모/자식간 상속아래)
2. 익명 구현객체 (인터페이스 구현)

익명 자식객체 생성 방법에는 아래 3가지방법으로 구현할 수 있습니다.
1. 필드의 초기값
2. 로컬변수의 초기값
3. 매개변수의 매개값

익명클래스 생성방법
=> 본인의 클래스 이름은 없다. 그러나 상속받을 부모클래스 or 인터페이스에 대해서는 어딘가에 명시해주어야 한다.
 익명클래스 생성시에는 상속받을 부모클래스 또는 구현할 인터페이스 이름과 함께 선언하며,
 클래스정의 및 객체생성의 두단계를 거치지 않고,
 new 연산자로 객체 생성과 동시에 {  } 안에 아래와 같은 형식으로 클래스 내용을 정의합니다.

부모클래스이름 변수명 = new 부모클래스 이름 {  ... 내용 구현 ...  };
인터페이스이름 변수명 = new 인터페이스 이름 {  ... 내용 구현 ...  };

★ 상속받은 클래스를 별도의 java파일을 통해 클래스를 만드는 것이 아닌 익명의 클래스를 만들어 사용하는 것이다.
★ 추상 클래스나 인터페이스를 간단히 구현하기 위해서 익명 클래스를 많이 사용한다.
